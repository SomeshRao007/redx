# Workout App — Project Instructions

## Keep README in sync after code changes

At the end of any session in which you modified code in this repo — files under `src/`, `functions/`, `migrations/`, `scripts/`, or root config (`package.json`, `wrangler.toml`, `vite.config.ts`, `tsconfig*.json`) — and **before** you write your final wrap-up summary, delegate a README refresh to the **`doc-updater`** subagent (it runs on Haiku 4.5). Then mention the README update in your wrap-up.

Skip this when the session changed no code (docs-only, read-only exploration, planning, or discussion). The `doc-updater` subagent edits only `README.md` and runs non-interactively.

To co-author a longer document interactively (proposal, spec, decision doc), use the `doc-coauthoring` skill instead — that one is hands-on and runs on the main session model, not Haiku.
